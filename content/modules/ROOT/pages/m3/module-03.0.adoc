=== {streams}によるStream Processing

Kafkaベースのアプリケーションが複雑化するにつれて、単にメッセージを送受信するだけでなく、より多くのことを行う必要が出てきます。多くの場合、アプリケーションは単にデータを他の場所に転送するのではなく、リアルタイムでデータを分析、集約、または反応する必要があります。そこで登場するのが、Kafka StreamsによるStream Processingです。

Kafka Streamsは、Apache Kafka上でリアルタイムのデータ処理とイベント駆動型アプリケーションを構築するための、軽量なクライアント側ライブラリです。

データを外部の処理クラスタに移動するのではなく、アプリケーション自体がstream processorになります。Kafka topicから読み取り、変換、マッピング、フィルタリング、結合、集約を実行し、結果をKafkaまたはローカルのstate storesに書き戻します。

=== アクティビティ概要
このモジュールでは、Kafka Streams APIを使用してeコマースWebアプリケーションからのユーザーアクティビティデータを処理する、実世界のストリーミングアプリケーションを探索します。

Globex Webアプリケーションが、前のモジュールでWebサイト上の製品に「いいね」をすることで生成したユーザーアクティビティに基づいて、Kafka Streams APIを使用してWebサイト上で最も人気のある製品のリストを構築する方法を分析しましょう。
