== Schema Registryへのスキーマの追加


このセクションでは、Schema Registryに新しいスキーマを追加します。後ほど、ProducerとConsumerがこのスキーマを使用してメッセージを送受信します。

// * First let's create a topic that we can use for this section. Run the following command in the *upper terminal*.
// +
// [source,sh,role="execute",subs=attributes+]
// ----
// kafka-topics.sh --bootstrap-server $BOOTSTRAP \
//   --create --topic schema-demo --partitions 3 --replication-factor 1
// ----

// NOTE: If you logged out of the toolbox use the following command to log back in: `oc rsh -n kafka-{user_name} kafka-toolbox`

* リンクをクリックして{registry} UIに移動します: https://service-registry-ui-{user_name}.{openshift_cluster_subdomain}[Schema Registry, window="_schema-registry"]。

* `Explore` タブをクリックします
+
image::m5/explore-tab.png[]

* 以下の画像のように、Schema Registry UIのExploreページが表示されます。
+
image::m5/schema-registry-ui-explore.png[]

* `Create Group` ボタンをクリックします。
+
image::m5/create-group-registry.png[]

* グループ名に `my-group` と入力し、createボタンを押します。
+
image::m5/create-mygroup.png[]

* `Create Artifact` ボタンをクリックして新しいスキーマを作成します。
+
image::m5/create-artifact.png[]


* Artifact Idに `schema-demo` と入力し、スキーマタイプとしてAvroを選択します。Nextをクリックします。
+
image::m5/artifact-coordinates.png[]

[NOTE]
====
次のステップで `schema-demo` という名前のトピックも作成します。ただし、Artifact Idとトピック名は常に同じである必要はありません。今回のケースでは、SimpleTopicIdStrategyを使用しているため、同じ名前を使用しました。使用できる異なるストラテジーがあります。詳細については link:https://docs.redhat.com/en/documentation/red_hat_build_of_apicurio_registry/3.1/html/apicurio_registry_user_guide/using-kafka-client-serdes_registry#registry-serdes-concepts-strategy_registry[こちら] で確認できます。
====

* 次の画面のすべてのフィールドはオプションです。空白のままにしてNextをクリックします。
+
image::m5/artifact-metadata.png[]

* 次の画面のcontentsセクションで、以下のスキーマをコピー&ペーストします。Version numberフィールドは空白のままにします。これは顧客レコード用のシンプルなAvroスキーマです。
+
[source,avro,role="execute",subs=attributes+]
----
{
  "type": "record",
  "name": "Customer",
  "namespace": "com.example.avro",
  "doc": "A sample Avro schema for a customer record",
  "fields": [
    {"name": "customer_id", "type": "string", "doc": "Unique identifier for the customer"},
    {"name": "first_name", "type": "string", "doc": "Customer's first name"},
    {"name": "last_name", "type": "string", "doc": "Customer's last name"},
    {"name": "email", "type": ["null", "string"], "default": null, "doc": "Email address (optional)"},
    {"name": "age", "type": ["null", "int"], "default": null, "doc": "Customer age (optional)"},
    {"name": "is_active", "type": "boolean", "default": true, "doc": "Whether the customer is active"},
    {"name": "signup_date", "type": {"type": "long", "logicalType": "timestamp-millis"}, "doc": "Timestamp of when the customer signed up"}
  ]
}
----

* Nextをクリックします。
+
image::m5/create-schema.png[]

* これは顧客データを含むメッセージのフィールドのルールを定義するシンプルなスキーマです。Avroスキーマについての詳細は link:https://avro.apache.org/[こちら] で確認できます。

* 最終画面でフィールドを空白のままにしてCreateをクリックします。
+
image::m5/create-schema-final.png[]

* Artifactの作成には数秒かかります。その後、以下の画像のように *Schema Registry UI* にスキーマが表示されます。
+
image::m5/schema-created.png[]

* 素晴らしい!Schema Registryに新しいスキーマを正常に追加できました。
