== Module 5: Schema Registry
{m5}

=== スキーマとは?
スキーマはデータ型の構造を形式的に記述したものです。スキーマを使用することで、実行時にデータを検証したり、Kafkaトピックから送受信されるデータをserialize/deserializeしたりできます。

=== なぜスキーマを使うのか?
スキーマを使用することで、以下のような複数のメリットが得られます:

* *データ検証:* 送受信されるデータが期待される構造に準拠していることを保証します。
* *Data serialization/deserialization:* データの効率的なserializationとdeserializationを可能にし、メッセージサイズを削減してパフォーマンスを向上させます。
* *スキーマ進化:* 既存のデータとの互換性を維持しながら、時間の経過とともにスキーマを変更できるようにします。
* *相互運用性:* 共通のスキーマを使用することで、異なるシステム間の通信を可能にします。



=== Schema Registry
Schema Registryは、イベント駆動アーキテクチャやAPIアーキテクチャ全体で標準イベントスキーマとAPI設計を共有するためのデータストアです。Registryを使用すると、データの構造をクライアントアプリケーションから分離し、REST interfaceを使用して実行時にデータ型を共有・管理できます。

ProducerやConsumerなどのクライアントアプリケーションは、再デプロイすることなく、実行時に最新のスキーマの更新をApicurio Registryに動的にpushしたり、pullしたりできます。開発チームは、本番環境にすでにデプロイされているサービスに必要な既存のスキーマをRegistryに問い合わせたり、開発中の新しいサービスに必要な新しいスキーマを登録したりできます。

=== Red Hat build of Apicurio Registry
Red Hatは、オープンソースのApicurio Registryプロジェクトに基づいたRed Hat build of Apicurio Registryを提供しています。クライアントアプリケーションはregistry URLを使用してserialization/deserializationと実行時の互換性チェックのためにスキーマを取得します。Webコンソールとオプションの検証/互換性ルールが、スキーマ進化の管理と適用を支援します。

=== Apicurio Registryの機能
* Apache Avro、JSON Schema、Google Protobuf、AsyncAPI、OpenAPIなどの標準イベントスキーマとAPI仕様のための複数のペイロード形式。
* Apache KafkaまたはPostgreSQLデータベースにコンテンツを保存するためのプラグ可能なApicurio Registryストレージオプション。
* Apicurio Registryコンテンツが時間の経過とともにどのように進化するかを管理するための、コンテンツ検証、互換性、整合性のルール。
* Webコンソール、REST API、コマンドライン、Mavenプラグイン、または言語SDKを使用したApicurio Registryコンテンツ管理。
* 外部システム向けのKafka Connectとの統合を含む、完全なApache Kafka schema registryサポート。
* 実行時にメッセージ型を検証するためのKafka client serializers/deserializers (SerDes)。
* 低メモリフットプリントと高速なデプロイ時間のためのCloud-native Quarkus Javaランタイム。
* OpenShift/Kubernetes上のApicurio RegistryのOperatorベースのインストール。
* Red Hat build of Keycloakを使用したOpenID Connect (OIDC)認証。

=== アクティビティ概要
このモジュールでは、Schema Registryを使用してKafkaトピックのスキーマを管理する方法を学びます。Kafkaトピック用の新しいスキーマを作成し、Schema Registryに登録し、そのスキーマをProducerとConsumerアプリケーションで使用してメッセージを送受信します。また、Schema Registryの互換性ルールについても簡単に学習します。
