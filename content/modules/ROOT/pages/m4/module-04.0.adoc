=== Change Data Capture

Change Data Capture (CDC) は、データベース内のデータに加えられた変更を識別してキャプチャし、それらの変更をリアルタイムでダウンストリームのプロセスやシステムに配信し、変更に対して反応できるようにするプロセスを指す、よく知られたソフトウェアパターンです。

CDCの典型的なユースケースには、データレプリケーション（あるデータストアから別のデータストアへのデータ移動）、分析システムやデータウェアハウスへのデータ投入、サービス間の疎結合なデータ伝播（1つのサービスがデータを所有し、他のサービスはそのデータのローカルで最適化されたビューを保持する）などがあります。このラボは後者のユースケースの例です。

オープンソースの世界では、https://debezium.io[Debezium] がCDCのデファクトスタンダードです。Debeziumはデータベースのトランザクションログを追跡することで、各データベーステーブル内の行レベルの変更を記録します。キャプチャされた変更は _change event stream_ に変換されます。

最も一般的に、DebeziumはKafka Connect上のconnectorとしてデプロイされます。Kafka Connectは、source connector（Kafkaにレコードを送信）とsink connector（KafkaトピックからレコードをKafkaトピックから他のシステムへ伝播）を実装して運用するためのフレームワークおよびランタイムです。Debeziumはsource connectorであり、change event streamをKafkaトピックに公開します。通常、監視対象のデータベーステーブルごとに1つのトピックが作成されます。

=== アクティビティの概要
このモジュールでは、Globex retailアプリケーションデータベースからの変更をキャプチャしてKafkaトピックに公開するDebezium connectorをデプロイします。次に、change eventが正しく公開されていることを確認します。また、Debeziumによって生成されたchange eventを他のアプリケーションがどのように使用してイベント駆動型アプリケーションを構築できるかを確認します。
